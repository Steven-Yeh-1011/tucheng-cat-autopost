# Meta 新版系統建立應用程式指南

Meta 已推出新的應用程式建立系統，提供更簡化的流程和更好的使用體驗。本指南專門針對新版系統。

## 新版系統的優勢

1. **可自訂的使用案例** - 根據您的需求選擇適合的使用案例
2. **新主控板** - 更直觀的介面設計
3. **簡化的權限設定** - 不需要從一長串權限和產品中挑選
4. **引導式設定** - 系統會引導您完成必要的設定步驟

---

## 步驟 1: 開始建立應用程式

### 1.1 登入並進入建立頁面

1. 前往：https://developers.facebook.com/
2. 使用您的 Facebook 帳號登入
3. 找到「應用程式」選單：
   - 在頁面頂部導航列中，點擊「**應用程式**」（Apps）
   - 或直接前往：https://developers.facebook.com/apps/
4. 點擊「建立應用程式」或「Create App」按鈕
   - 位置：應用程式列表頁面的右上角，或空狀態頁面的中央
   - 或直接前往：https://developers.facebook.com/apps/creation/

### 1.2 處理新系統彈窗

當您看到「你可以透過新方式使用 Meta 建立應用程式」的彈窗時：

1. **閱讀說明**：
   - 新系統提供可自訂的使用案例
   - 新的主控板設計
   - 更簡化的資料存取權限要求方式
   - 不需要從一長串權限和產品中挑選

2. **選擇操作**：
   - 如果想了解更多：點擊「深入瞭解如何在我們的新系統中建立應用程式」
   - 如果準備就緒：點擊「建立應用程式」（藍色按鈕）
   - 如果想返回：點擊「返回」

3. **開始使用導覽**：
   - 點擊「建立應用程式」後，系統會開始新主控板的使用導覽

---

## 步驟 2: 填寫應用程式詳細資料

### 2.1 應用程式名稱

- **欄位**: 「應用程式名稱」或「App Name」
- **說明**: 這個應用程式名稱會顯示在您的應用程式列表中
- **建議**: 使用清楚的名稱，例如：`土城貓舍自動發文系統`
- **限制**: 最多 30 個字元

### 2.2 應用程式聯絡電子郵件地址

- **欄位**: 「應用程式聯絡電子郵件地址」或「App Contact Email Address」
- **說明**: Meta 會使用此電子郵件地址與您聯繫，或在帳號被移除或入侵時用於復原步驟
- **範例**: `s0919708515@yahoo.com`
- **重要**: 請使用您經常檢查的 Email

### 2.3 繼續下一步

填寫完成後，點擊右下角的「繼續」或「Continue」按鈕。

---

## 步驟 3: 選擇使用案例

### 3.1 使用案例選項

新系統會根據常見的使用場景提供預設的使用案例，例如：

- **Business** - 商業應用
- **Social Media Management** - 社群媒體管理（適合自動發文系統）
- **E-commerce** - 電商應用
- **Other** - 其他用途

### 3.2 推薦選擇

對於「土城貓舍自動發文系統」，建議選擇：

1. **Social Media Management**（如果可用）
   - 系統會自動設定相關的權限和產品
   - 包含 Facebook 和 Instagram 發佈功能

2. **Business**（如果沒有 Social Media Management）
   - 然後手動新增需要的產品和權限

3. **Other**（如果需要完全自訂）
   - 可以完全自訂所有設定

---

## 步驟 4: 新主控板導覽

### 4.1 主控板功能

新主控板提供：

- **總覽** - 應用程式基本資訊和狀態
- **產品** - 已新增的產品列表
- **權限** - 資料存取權限管理
- **設定** - 應用程式詳細設定

### 4.2 完成必要設定

系統會引導您完成：

1. **基本設定**
   - App ID 和 App Secret
   - 平台設定（Web、iOS、Android）

2. **產品設定**
   - 根據使用案例自動建議的產品
   - 可以手動新增或移除

3. **權限設定**
   - 根據使用案例自動建議的權限
   - 簡化的權限申請流程

---

## 步驟 5: 設定 Facebook 和 Instagram 發佈

### 5.1 確認產品已新增

在新主控板中確認以下產品已新增：

- ✅ **Facebook Login** - 用於取得 Access Token
- ✅ **Instagram Graph API** - 用於發佈到 Instagram
- ✅ **Pages API** - 用於管理粉絲專頁

如果沒有，請手動新增。

### 5.2 設定 OAuth 重新導向 URI

1. 前往「Facebook Login」→「Settings」或「設定」
2. 在「Valid OAuth Redirect URIs」中新增：
   ```
   https://tucheng-cat-autopost.onrender.com
   ```
3. 點擊「Save Changes」或「儲存變更」

### 5.3 設定應用程式網域

1. 前往「Settings」→「Basic」或「設定」→「基本」
2. 在「App Domains」中新增：
   ```
   onrender.com
   ```
3. 點擊「Save Changes」

---

## 步驟 6: 取得必要的 ID 和 Token

### 6.1 取得 App ID 和 App Secret

在新主控板中：

1. 前往「Settings」→「Basic」或「設定」→「基本」
2. 找到：
   - **App ID** - 複製此 ID
   - **App Secret** - 點擊「Show」並輸入密碼後複製

### 6.2 取得 Page ID

使用 Graph API Explorer 或從粉絲專頁設定中取得（參考主指南）

### 6.3 取得 Instagram Account ID

使用 Graph API Explorer 查詢（參考主指南）

### 6.4 取得 Access Token

使用 Graph API Explorer 或 Page Access Token（參考主指南）

---

## 步驟 7: 申請權限（如果需要）

### 7.1 檢查已自動設定的權限

新系統可能已根據使用案例自動設定一些權限，檢查：

- `pages_manage_posts` - 發佈貼文
- `pages_read_engagement` - 讀取互動
- `instagram_basic` - Instagram 基本功能
- `instagram_content_publish` - 發佈到 Instagram

### 7.2 申請額外權限

如果缺少必要權限：

1. 前往「App Review」→「Permissions and Features」或「應用程式審查」→「權限和功能」
2. 搜尋需要的權限
3. 點擊「Request」或「申請」
4. 填寫使用案例說明
5. 提交審核（測試環境可先跳過）

## 新版系統 vs 傳統系統對照

| 功能 | 新版系統 | 傳統系統 |
|------|---------|---------|
| 建立流程 | 引導式，根據使用案例 | 手動選擇類型 |
| 產品設定 | 自動建議 | 手動新增 |
| 權限設定 | 簡化流程 | 從列表中選擇 |
| 主控板 | 新設計 | 傳統設計 |
| 使用導覽 | 有 | 無 |

## 常見問題

### Q: 我可以切換回傳統系統嗎？

A: 一旦使用新系統建立應用程式，就會使用新主控板。但功能基本相同，只是介面不同。

### Q: 新系統會自動設定所有需要的權限嗎？

A: 不會完全自動，但會根據您選擇的使用案例建議相關權限，您仍需要確認和申請。

### Q: 如何知道我的應用程式是用新系統還是傳統系統建立的？

A: 如果看到新的主控板設計和使用導覽，就是新系統。傳統系統的介面較為簡單。

### Q: 新系統的 App ID 和 App Secret 格式有改變嗎？

A: 沒有，格式完全相同，只是取得的位置可能不同。
