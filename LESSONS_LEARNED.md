## 🔴 嚴重錯誤 4: 環境變數未正確載入

### 問題描述

更新 .env 檔案後，腳本仍使用舊的環境變數。

### 根本原因

- ❌ **PowerShell 環境變數緩存**
- ❌ **沒有重新載入環境變數**
- ❌ **腳本沒有明確指定 .env 檔案路徑**

### 防範措施（絕對不能再執行）

#### ✅ 環境變數載入規則

1. **腳本必須明確指定 .env 路徑**
   ```typescript
   // ✅ 正確做法
   const projectRoot = path.resolve(__dirname, '../../..');
   dotenv.config({ path: path.join(projectRoot, '.env') });
   
   // ❌ 錯誤做法
   dotenv.config(); // 可能讀取到錯誤位置的 .env
   ```

2. **更新 .env 後必須重新載入**
   - PowerShell 中需要重新載入環境變數
   - 或者重啟終端機

#### ✅ 規則

1. **永遠**在腳本中明確指定 .env 檔案路徑
2. **永遠**從專案根目錄讀取 .env
3. **更新 .env 後**必須確認腳本讀取到新值
